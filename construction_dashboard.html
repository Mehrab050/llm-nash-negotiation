Project Overview

This dashboard is designed so that a viewer can understand the entire operating logic without opening the Python source code. It combines scenario KPIs, the decision pipeline, an operational timeline, and an explanation viewer for each negotiation event.

How the System Operates

This is the core operating sequence of the project.

1

Tasks Become Ready

Robots such as Frame, Roof, and Electrical move into a ready state when dependency thresholds are satisfied.

2

Resource Conflict Appears

Two or more tasks request the same scarce resource like the crane or work zone at the same simulation minute.

3

Nash Score Is Computed

The system scores each candidate using criticality, downstream pressure, waiting time, and schedule impact.

4

Winner Is Chosen

The highest-priority task receives the contested resource. Others are deferred and the schedule is updated.

5

LLM Explains the Decision

The LLM does not need to choose the winner. It can explain why the chosen allocation protects the critical path.
